Vue历练之路二---Vue的基础语法

Vue起步

<div id="app">Hello,Vue</div>

<script>
    var app = new Vue({
        el: '#app',
        data:{},
        ...
    });
</script>

el：el指定的容器会交由Vue进行管理，Vue内部会解析容器中的一些固定语法，完成数据的展示。

data：data用于指定数据

在Vue对象中，我们还可以指定很多的内容，后续再做补充。

Vue的模板语法

Mustache语法

Mustache标签将会被替代为对应数据对象上message的值。无论何时，绑定的数据对象上messagee的值发生了改变，插值处的内容都会更新。格式: {{}}

HTML：

<div id="app">
    <h2>Mustache语法</h2>
    <span>{{message}}</span>
</div>

JavaScript:

var mv = new Vue({
    el: '#app',
    data:{
        message: 'Mustache'
    }
});

效果:

v-once指令

​ 通过使用 v-once 指令，可以进行一次性的赋值。在修改绑定数据时，该视图不会改变。

HTML：

<div id="app">
    <h2>v-once指令</h2>
    <span v-once>{{once}}</span>
</div>

JavaScript:

var mv = new Vue({
    el: '#app',
    data:{
        once: 'v-once'
    }
});

效果:

v-cloak指令

​ Mustache语法在前端数据量大，数据处理较慢时会出现语法代码闪烁的情况。v-cloak指令可以解决闪烁问题。

HTML:

<div id="app">
    <h2>v-cloak指令，可以解决Mustache闪烁问题</h2>
    <span v-cloak>{{cloak}}</span>
</div>

CSS:

[v-cloak] {
 display: none !important;
}

JavaScript:

var mv = new Vue({
    el: '#app',
    data:{
        cloak: 'cloak'
    }
});

​ v-cloak实现原理就是在页面加载时先使标签不显示，在标签加载完成后由vue将display属性删除。

v-text指令

​ v-text渲染的标签内容不会被解析，会被作为字符文本渲染，并且没有闪烁问题。

HTML:

<div id="app">
    <h2>v-text指令，用于填充纯文本内容</h2>
    <span v-text="text"></span>
</div>

JavaScript:

var mv = new Vue({
    el: '#app',
    data:{
        text: 'v-text'
    }
});

效果:

v-html指令

​ v-html渲染的标签内容会被当做html渲染。

HTML：

<div id="app">
   <h2>v-html指令,用于输出HTML内容</h2>
   <span v-html="html"></span>
</div>

JavaScript:

var mv = new Vue({
    el: '#app',
    data:{
        text: 'v-text'
    }
});

效果：

v-pre指令

​ v-pre指令可以忽略语法解析，跳过这个元素和它的子元素的编译过程。一些静态的内容不需要编辑加这个指令可以加快编辑。

HTML:

<div id="app">
    <h2>v-pre指令,忽略语法解析</h2>
    <span v-pre>{{pre}}</span>
</div>

JavaScript:

var mv = new Vue({
    el: '#app',
    data:{
        pre: 'pre'
    }
});

效果:
